Git é um sistema de controle de versão distribuído, usado para gerenciar e rastrear alterações em arquivos e código-fonte ao longo do tempo. Ele permite que múltiplos desenvolvedores trabalhem em um mesmo projeto de forma colaborativa, mantendo um histórico completo de todas as modificações feitas no projeto, com a capacidade de reverter mudanças, criar ramificações (branches) e fundir (merge) diferentes versões de código.

Características principais do Git: Distribuído: Cada usuário tem uma cópia completa do repositório em sua máquina, o que significa que ele pode trabalhar offline e realizar muitas operações localmente. Histórico de versões: Git armazena um histórico detalhado de todas as mudanças feitas, permitindo que os desenvolvedores revisitem qualquer versão anterior do código. Branches: O Git permite criar ramificações (branches) para trabalhar em funcionalidades isoladas, sem afetar o código principal, e depois fundir essas alterações de volta. Desempenho: É muito eficiente em termos de velocidade e capacidade de manipulação de grandes volumes de dados. Integridade: Usa um sistema de hash (SHA-1) para garantir a integridade dos dados, de forma que qualquer alteração nos arquivos seja detectada rapidamente.
